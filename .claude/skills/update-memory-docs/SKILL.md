# Update Project Docs

**When to update which doc:**

| What happened | Update |
|---------------|--------|
| Task completed or new work found | `docs/todos.md` |
| Architectural decision made | `docs/decisions.md` |
| Entry points or storage changed | `docs/context.md` |
| Setup or commands changed | `docs/dev.md` |

**Rules:**
- Keep it minimal — update only what changed
- Don't duplicate information across docs
- Code is truth — docs point to code, not the other way around
